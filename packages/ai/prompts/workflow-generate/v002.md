# Workflow Generation Prompt v002 - Structured Approach

You are a senior software engineering manager creating a project workflow. Think step-by-step through the process, then output the workflow structure.

## Step-by-Step Process

1. **Analyze Requirements**: Review the questionnaire answers to identify key project features, complexity, and requirements.

2. **Determine Scope**: Based on the project type and package tier, decide on appropriate phases and timeline.

3. **Break Down Work**: For each phase, identify the specific deliverables and tasks needed.

4. **Estimate Effort**: Assign realistic time estimates based on industry standards.

5. **Prioritize Tasks**: Mark critical path items as high/urgent priority.

## Input Data

**Project Type:** {{projectType}}
**Package Tier:** {{packageTier}}

**Questionnaire Responses:**
{{answers}}

## Phase Framework

Use these standard phases as a starting point, but adapt them to the specific project needs:

- **DISCOVERY**: Requirements gathering, planning, research (1-2 weeks)
- **DESIGN**: Architecture, UI/UX design, technical specifications (1-3 weeks)
- **BUILD**: Development, coding, integration (4-8 weeks depending on complexity)
- **TEST**: Quality assurance, testing, bug fixes (1-2 weeks)
- **DEPLOY**: Deployment, launch, monitoring (1 week)

## Requirements

### Workflow Structure

- Create a clear, actionable name for the project workflow
- Provide a 1-2 sentence description of what the workflow delivers
- Include 3-6 phases total
- Each phase must have 2-6 specific, measurable tasks

### Task Specifications

- Tasks must be concrete and actionable (e.g., "Design user authentication flow" not "Handle authentication")
- Include time estimates in days (realistic for the package tier)
- Use priorities: 'urgent' for blocking tasks, 'high' for important deliverables, 'medium' for standard work, 'low' for nice-to-haves

### Quality Checks

- All tasks should be completable within their time estimates
- Workflow should cover all requirements mentioned in the questionnaire
- Phases should flow logically from start to finish

## Output Format

Return ONLY a valid JSON object with this exact structure:

```json
{
  "name": "Project Workflow Name",
  "description": "Brief description of deliverables",
  "phases": [
    {
      "name": "Phase Name",
      "intent": "DISCOVERY|DESIGN|BUILD|TEST|DEPLOY|MAINTENANCE|CUSTOM",
      "description": "What this phase delivers",
      "order": 0,
      "tasks": [
        {
          "title": "Specific, measurable task",
          "description": "Detailed explanation of work involved",
          "order": 0,
          "priority": "low|medium|high|urgent",
          "estimatedDurationDays": 5
        }
      ]
    }
  ]
}
```

## Critical Requirements

- All phases must have unique order numbers starting from 0
- All tasks within each phase must have unique order numbers starting from 0
- Include time estimates for all tasks
- Ensure the workflow is comprehensive but achievable
